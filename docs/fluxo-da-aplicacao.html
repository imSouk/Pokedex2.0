Pokédex 2.0

Documentação do fluxo da aplicação — do clique do usuário à resposta da API

Stack: React 18 + React Router 7 + Vite 5 + Tailwind CSS 3

API: PokéAPI v2 (pública, sem autenticação)

Rotas: / (lista) e /Detalhes?name=<pokemon> (detalhe)

1. Fluxo Geral da Aplicação

1
Usuário acessa /
O componente Body monta e dispara useEffect.
↓
2
fetchPokemon("", callback)
Chamada à PokéAPI para carregar os 151 Pokémon de Kanto.
↓
3
Response é transformada
Cada Pokémon vira um objeto { name, id, type, sprite }.
↓
4
Grid de cards é renderizado
19 cards por página, com setas "Prev" / "Next".
↓
5
Usuário digita + Enter
Busca por nome ou ID: fetchPokemon("pikachu", setState).
↓
6
Usuário clica em um card
Navega para /Detalhes?name=<pokemon>.
↓
7
Detalhes monta e chama fetchinfoPokemon
Busca dados completos do Pokémon na PokéAPI.
↓
8
Tela de detalhes é renderizada
Header colorido, sprite, stats, habilidades.

2. Rota / — Lista de Pokémon

2.1. Primeiro carregamento (todos os 151 Pokémon)

Requisição: GET https://pokeapi.co/api/v2/pokemon?limit=151

Resposta da API (lista de resultados):

{
  "results": [
    { "name": "bulbasaur", "url": "https://pokeapi.co/api/v2/pokemon/1/" },
    { "name": "ivysaur",   "url": "https://pokeapi.co/api/v2/pokemon/2/" }
    // ... até 151
  ]
}

Em seguida, para cada url da lista, o código faz uma segunda requisição individual:

Requisição (151x): GET https://pokeapi.co/api/v2/pokemon/{id}
Ex: GET https://pokeapi.co/api/v2/pokemon/1/

Resposta individual (ex: bulbasaur):

{
  "id": 1,
  "name": "bulbasaur",
  "types": [
    { "slot": 1, "type": { "name": "grass" } },
    { "slot": 2, "type": { "name": "poison" } }
  ],
  "sprites": {
    "other": {
      "dream_world": {
        "front_default": "https://raw.githubusercontent.com/.../1.svg"
      }
    }
  }
}

Dado transformado para o estado do componente:

[
  {
    "name": "bulbasaur",
    "id": 1,
    "type": "grass",         // só o primeiro tipo
    "sprite": "https://raw.githubusercontent.com/.../1.svg"
  }
  // ... até 151
]

2.2. Busca por nome ou ID (campo de busca)

Requisição: GET https://pokeapi.co/api/v2/pokemon/{nomeOuId}
Ex: GET https://pokeapi.co/api/v2/pokemon/pikachu
⚠️ Caso de erro (404 — Pokémon não encontrado):
  1. Exibe alert("pokemon Não encontrado")
  2. Recarrega os 151 Pokémon automaticamente (fallback)
  3. Retorna à lista completa

Resposta de sucesso: mesmo formato do item individual acima.

Dado transformado (array com 1 elemento):

[
  {
    "name": "pikachu",
    "id": 25,
    "type": "electric",
    "sprite": "https://raw.githubusercontent.com/.../25.svg"
  }
]

3. Rota /Detalhes?name=<pokemon> — Página de Detalhes

Requisição: GET https://pokeapi.co/api/v2/pokemon/{name}
Ex: GET https://pokeapi.co/api/v2/pokemon/pikachu

6. Resumo das Chamadas à API

FunçãoEndpointQuandoResposta usada para
fetchPokemon("") GET /pokemon?limit=151 + 151x GET /pokemon/{id} Ao montar a página inicial Lista de cards com {name, id, type, sprite}
fetchPokemon(nome) GET /pokemon/{nome} Ao pressionar Enter no campo de busca Card único do Pokémon pesquisado
fetchinfoPokemon(nome) GET /pokemon/{nome} Ao entrar na página de detalhes Dados completos: stats, habilidades, tipos, sprite
